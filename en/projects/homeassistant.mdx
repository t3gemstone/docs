## 1. Introduction

Today, controlling your home automation data and securing your smart devices is more important than ever.
You may want to manage your lights, security cameras, smart plugs, and sensors from a central system
without depending on large cloud providers such as Amazon, Google, Xiaomi, or Tuya.
To do this, you need to host your own smart home server on your own device and control your devices over the local network.
However, running such a hub at home brings technical challenges such as making devices from different brands communicate
with each other, the difficulty of providing secure access from the outside world, and keeping the system up and running at all times.
These problems are usually solved with complex configurations and expensive hardware.
The T3 Gemstone O1 project, with its Home Assistant integration, solves these problems in an easy, low-cost, and secure way.


## 2. Installing Home Assistant

Home Assistant is an open-source smart home and automation platform. It offers features such as monitoring sensor data,
controlling lights, plugs, and other devices, creating automated scenarios, and access from mobile devices, all in a single system.

After establishing a [terminal connection](/en/quickstart/#2-explore-gemstone) to the Gemstone development board,
first start the Docker installation with the following command:

```bash
sudo curl -fsSL https://get.docker.com | sudo sh
```

Once the Docker installation is complete, you can install **Home Assistant**.

```bash
sudo apt install t3-gem-homeassistant
```

After the installation finishes, it may take 5-10 minutes for the Supervisor to download and start all services, depending on your internet speed.
You can monitor the status with the following command:

```bash
sudo docker logs -f hassio_supervisor
```

When you see that the `hassio_supervisor` and `homeassistant` services are in the `Up` (Running) state, you can access the interface from your browser:

`http://gemstone.local:8123` or `http://<your-gemstone-ip-address>:8123`

To access your system from outside your home, you can use the [Tailscale](/en/projects/cloud#3-accessing-from-the-outside-world) network, for which an installation guide is available, and connect securely to your system via your Tailscale IP address (`http://<tailscale-ip>:8123`).

In this step, you need to give the board a name.

<Frame caption="Completing the ESPHome Device Configuration">
  <img src="/images/homeassistant/en/esphome-device-wifi.png" alt="Step n" />
</Frame>

In this step, enter your router's SSID and password in the "Network Name" and "Password" fields.

```yaml
substitutions:
  LED_PIN: GPIO48      # RGB LED on the board
  BUTTON_PIN: GPIO0    # Boot Button
  
  # Pin Mappings
  D0: GPIO1
  D1: GPIO2
  D4: GPIO42
  D8: GPIO38
  D9: GPIO48
  D12: GPIO10
  D13: GPIO3
  D14: GPIO8
  # I2C
  SDA_PIN: GPIO47
  SCL_PIN: GPIO21
```
Then, copy the **pin definitions** made in this section to the bottom of the YAML file.

<Frame caption="Adding a Sensor to the ESPHome YAML File">
  <img src="/images/homeassistant/en/esphome-add-sensor.png" alt="Step y" />
</Frame>

Since a motion sensor will be connected in this example, paste the following definitions at the bottom of the YAML file as shown above.

```yaml
binary_sensor:
  - platform: gpio
    pin: 
      number: GPIO42
      mode: INPUT
      inverted: false
    name: "PIR Sensor"
    device_class: motion
    filters:
      - delayed_on: 1s
```


<Note>
 The motion sensor is connected to the **D4 pin (GPIO42)**. If you connected it to a different pin, you need to update the **`number`** field according to the relevant pin.
 For the required pin assignments, you can refer to the [Deneyap Kart Pin Guide](https://docs.deneyapkart.org/en/Development_Boards) page.
</Note>

Click the "Install" button in the bottom right corner. This step completes the configuration.

## 5. Frequently Asked Questions

**I am getting an "Unsupported System" warning after installation. Why?**
> This warning may appear when an operating system other than Home Assistant OS is used, or when other manually running services (such as Nextcloud or Tailscale) exist on the system. It does not prevent the smart home system from running stably; it is for informational purposes only.

**When will the Add-on store appear?**
> If you open the interface right after the installation completes, the Add-ons menu may not be visible. Once the system finishes the necessary configuration in the background (usually within 5-10 minutes), the "Settings -> Add-ons" tab will become active automatically.
