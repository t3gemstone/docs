## 1. Introduction

Today, the control of home automation data and the security of smart devices are more important than ever. You may want to manage your lighting, security cameras, smart plugs, and sensors in a centralized system without relying on major cloud providers such as Amazon, Google, Xiaomi, or Tuya. To achieve this, you need to host your own smart home server on your own device and control your devices over the local network.

However, running such a hub in your home brings technical challenges, including enabling devices from different brands to communicate with each other, the inability to provide secure remote access, and keeping the system continuously operational. These problems are often solved with complex configurations and high-cost hardware. The T3 Gemstone O1 project, with its Home Assistant integration, offers an easy, low-cost, and secure solution to these problems.

## 2. Home Assistant Installation

Home Assistant is an open-source smart home and automation platform. It offers features such as monitoring sensor data, controlling lights, plugs, and other devices, creating automated scenarios, and providing mobile access — all within a single system.

After establishing a [terminal connection](/en/quickstart/#2-explore-gemstone) to the Gemstone development board, start the Docker installation with the following command:

```bash
sudo curl -fsSL https://get.docker.com | sudo sh
```

Docker installation complete. You can now install **Home Assistant**.

```bash
sudo apt install t3-gem-homeassistant
```

After the installation is complete, it may take 5–10 minutes depending on your internet speed for Supervisor to download and start all services. You can monitor the status with the following command:

```bash
sudo docker logs -f hassio_supervisor
```

Once you see the `hassio_supervisor` and `homeassistant` services in the `Up` state on the screen, you can access the interface from your browser:

`http://gemstone.local:8123` veya `http://<your-gemstone-ip-address>:8123`

To access your system from outside your home network, you can use [Tailscale](/en/projects/cloud#3-access-from-the-public-internet), which includes a setup guide. You can securely connect to your system using your Tailscale IP address (`http://<tailscale-ip>:8123`).

### 4.2 Configuring ESPHome for Deneyap Kart

In the opened menu, click the "New Device" option.

<Frame caption="ESPHome Add New Device Screen">
  <img src="/images/homeassistant/esphome-add-device.png" alt="Add New Device" />
</Frame>

Click the **Continue** button to proceed.

<Frame caption="ESPHome Device Configuration Screen">
  <img src="/images/homeassistant/esphome-device-config.png" alt="Device Configuration" />
</Frame>

Click the **New Device Setup** option.

<Frame caption="ESPHome Device Configuration Screen">
  <img src="/images/homeassistant/esphome-device-config-2.png" alt="Device Configuration Step 2" />
</Frame>

In this step, you must assign a **name** to the board. Additionally, enter your modem's **SSID** and password in the **"Network Name"** and **"Password"** fields.

<Frame caption="ESPHome Device Type Selection Screen">
  <img src="/images/homeassistant/esphome-device-type.png" alt="Device Type Selection" />
</Frame>

In this step, select the **ESP32-S3** option.

<Frame caption="ESPHome Device Configuration Complete">
  <img src="/images/homeassistant/esphome-config-complete.png" alt="Configuration Complete" />
</Frame>

At this stage, click the **Skip** option. Then, from the **Deneyap Kart** section displayed in the background, click the **Edit** button.

<Frame caption="ESPHome YAML File Edit Screen">
  <img src="/images/homeassistant/esphome-yaml-edit.png" alt="YAML File Edit" />
</Frame>

In this step, you will see a **YAML** file.

- Under `esp32`, update the **`board`** field according to the Deneyap board being used. For **Deneyap Kart 1A V2**, delete the existing `board:` line and replace it with **`deneyapkart1Av2`**.
- Additionally, change the **`framework`** field so that it reads **`type: arduino`**.

<Frame caption="ESPHome YAML File GPIO Configuration">
  <img src="/images/homeassistant/esphome-gpio-setup.png" alt="GPIO Configuration" />
</Frame>

```yaml
substitutions:
  LED_PIN: GPIO48      # Onboard RGB LED
  BUTTON_PIN: GPIO0    # Boot Button
  
  # Pin Mappings
  D0: GPIO1
  D1: GPIO2
  D4: GPIO42
  D8: GPIO38
  D9: GPIO48
  D12: GPIO10
  D13: GPIO3
  D14: GPIO8
  # I2C
  SDA_PIN: GPIO47
  SCL_PIN: GPIO21
```
Then, the **pin mappings** created in this section are copied to the bottom of the YAML file.

<Frame caption="ESPHome Add Sensor to YAML File">
  <img src="/images/homeassistant/esphome-add-sensor.png" alt="Add Sensor" />
</Frame>

In this example, a motion sensor is being connected, so the following definitions are pasted at the bottom of the YAML file as shown in the figure above.

```yaml
binary_sensor:
  - platform: gpio
    pin: 
      number: GPIO42
      mode: INPUT
      inverted: false
    name: "PIR Sensor"
    device_class: motion
    filters:
      - delayed_on: 1s
```


<Note>
The motion sensor is connected to pin **D4 (GPIO42)**. If you connected it to a different pin, you need to update the **`number`** field accordingly. For required pin assignments, refer to the [Deneyap Kart Pin Guide](https://docs.deneyapkart.org/en/Development_Boards).
</Note>

Click the **Install** button located in the top right corner. This step completes the configuration.

## 5. Frequently Asked Questions

**I receive a "Unsupported System" warning after installation. Why?**
> This warning may appear when using an operating system other than Home Assistant OS or when there are other manually running services (such as Nextcloud or Tailscale) on the system. It does not prevent the smart home system from operating stably and is provided for informational purposes only.

**When will the Add-on store appear?**
> If you access the interface immediately after installation, the Add-ons menu may not be visible. Once the system completes the necessary background configurations (typically within 5–10 minutes), the **Settings -> Add-ons** tab will become available automatically.
